# Идеи по росту выручки

Бэклог продуктовых гипотез. Контекст — диагностика воронки 26.05.26:
визиты 1119/нед → free-видео 756 → **открыли оплату 13 (1,7%)** → оплатили 3.
Главный обрыв: человек получает бесплатное видео и уходит. Чиним конверсию free→оплата, НЕ трафик.

---

## 💡 Free без скачивания (gating загрузки) — приоритет

**Идея (от пользователя, 27.05.26):** бесплатно дать только **просмотр** готового видео,
а **скачивание / получение файла без watermark — только после оплаты**.

**Зачем:** сейчас 98% забирают бесплатный результат целиком и уходят — платить незачем.
Если файл нельзя унести, остаётся мотивация заплатить за «забрать своё видео».

**Как мерить:** цель `VIDEO_DOWNLOAD` (561354412) уже трекается в Метрике.
Смотрим связку `FREE_GEN_COMPLETED → PAYMENT_OPEN → PAYMENT_SUCCESS` до и после.

**Риск:** слишком жёсткий gating может убить вау-эффект и отпугнуть — держать просмотр
бесплатным и приятным, ограничивать только вынос файла. Сочетать с [[watermark_notice]]:
честно сообщать «у платной версии watermark'a не будет + можно скачать в HD».

**Где трогать:** Telegram-бот (`AI_Photo2Video_Bot`) — отдавать превью, не файл;
лендинг (`web-video-create`) — показывать плеер, кнопка «Скачать» → оплата.

## 💡 Бот в мессенджере MAX — новый канал

**Идея (от пользователя, 27.05.26):** зарегистрировать бота-оживлятор фото в мессенджере **MAX** (VK, РФ).

**Зачем:** целевой рынок — RU+СНГ, MAX активно растёт за счёт госпуша в РФ, конкуренция AI-ботов там
сейчас низкая → шанс на дешёвый ранний охват и трафик в обход дорогого Яндекс.Директа.

**Проверить перед стартом:** (1) есть ли у MAX Bot API и какие лимиты; (2) поддержка приёма платежей
внутри MAX (или вести на лендинг/ЮKassa); (3) платёжеспособность и размер аудитории MAX.

**Как мерить:** отдельный source-таг для MAX-юзеров, сравнивать конверсию free→оплата с Telegram/лендингом.

**Связь:** [[target_market]] (RU+СНГ), может переиспользовать логику Telegram-бота.
