Что должен сделать пользователь сам в браузере

Эти шаги Claude.ai в браузере может объяснить пользователю если он попросит, но не должна делать сама без подтверждения.

  1. myaccount.google.com/permissions → найти botisk-youtube-uploader (или среди «Проектов без названия» — то у которого скоупы youtube.upload + youtube.readonly + yt-analytics.readonly) → Remove Access
  2. В YouTube Studio убедиться что активен канал VideoAI (галочка ✓)
  3. Также: на странице YouTube Analytics API в Google Cloud → нажать ENABLE. Без этого yt-analytics.readonly скоуп не работает.
  4. В PowerShell: cd "C:\AI moy\test 1 moy"; & "C:\Users\Денис\AppData\Local\Programs\Python\Python312\python.exe" scripts\youtube_oauth_setup.py
  5. На Google OAuth странице — выбрать аккаунт. Должны быть оба варианта (Old Loft и VideoAI). Выбрать VideoAI.
  6. Дальше: Advanced → Continue → разрешить 3 scopes
  7. В консоли появится «refresh_token сохранён в .env» → сказать Claude Code «готово»

Гипотеза для следующей сессии

Прошлая гипотеза («переключение активного канала меняет API») не подтвердилась. Текущая гипотеза: prompt=consent select_account в OAuth URL + revoke предыдущего grant'a даст браузеру правильный «Выберите канал» экран Google, где будут оба варианта (включая brand-account VideoAI).
