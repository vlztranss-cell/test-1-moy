## Введение

В мае 2026 года я запустил сервис botisk.ru — оживление фотографий через AI. За 8 дней через пайплайн прошло 600+ генераций, и я готов поделиться **реальной production-статистикой** трёх ведущих video-AI моделей, а не маркетинговыми бенчмарками.

В статье — сравнение **Kling 2.5 (Kuaishou) vs Sora (OpenAI) vs Runway Gen-3**, **архитектура pipeline** (n8n + Kling + ffmpeg + PostgreSQL), и **подводные камни**, о которых не пишут в туториалах.

## Почему именно Kling, а не Sora

Sora и Runway недоступны напрямую из РФ. Можно через VPN + американскую карту, но это нестабильно и **в 5-10 раз дороже на единицу** ($0.50-$1.00 за видео vs $0.10 у Kling через PiAPI).

Kling 2.5 от Kuaishou (TikTok-конкурент) доступен через PiAPI (https://piapi.ai), оплата с любой карты, латентность 30-60 секунд на 5-секундное видео 1080p 9:16.

## Архитектура pipeline (production)

```
[Юзер на лендинге]
    ↓ загружает фото
[n8n: Web_Photo2Video_MVP]
    ↓ файл на freeimage.host (бесплатный CDN)
[PiAPI Kling 2.5]
    ↓ task_id (асинхронно)
[Polling status каждые 5 сек]
    ↓ ready
[Download MP4]
    ↓ ffmpeg overlay watermark + Pillow text PNG
[CDN — Caddy proxy /videos/]
    ↓ presigned URL
[Юзер скачивает]
```

**Ключевые тонкости:**

### 1. Service mode "public" для PAYG

PiAPI по умолчанию использует HYA-квоту (фиксированную ежемесячную). При запросе без `service_mode="public"` он жжёт квоту, а после её исчерпания — возвращает HTTP 429. Прячьте «public» в каждом запросе если работаете на pay-as-you-go.

```json
{
  "model": "kling",
  "task_type": "video_generation",
  "input": {
    "prompt": "natural movement, blink, smile",
    "negative_prompt": "blurry, distorted",
    "duration": 5,
    "aspect_ratio": "9:16",
    "image_url": "https://...",
    "version": "2.5",
    "cfg_scale": 0.5
  },
  "config": {"service_mode": "public"}
}
```

### 2. Content-policy: Kling отвергает ~10% фото

Через 3 дня production я столкнулся с тем, что **каждое восьмое** фото отклоняется китайским upstream-сервером Kling. Сообщение об ошибке:

```
图片上传被上游拒绝，请检查图片内容或格式后更换素材重试
```

Что отклоняет (по моим тестам):
- Логотипы и торговые марки (даже если они на фоне)
- Лица публичных людей (актёры, политики, в т.ч. русские)
- Контент 18+ (даже намёки)
- Очень тёмные / низкого качества фото
- Иногда — пожилые лица с морщинами (false-positive)

**Решение:** в UI заранее предупреждать пользователя что подходит, а что нет, и автоматически возвращать кредит при отказе.

### 3. Cloudflare блокирует Python-urllib

`api.piapi.ai` и `storage.theapi.app` блочат Python-urllib User-Agent с HTTP 403 (Cloudflare ботозащита). Нужно подменять UA на любой Browser-like:

```python
req = urllib.request.Request(url, headers={
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...",
    ...
})
```

Curl при этом работает — Cloudflare пропускает его без проблем. Это не фича, это **специфика их WAF rule** для Python-Requests.

### 4. Финальный рендеринг — ffmpeg static без drawtext

Я использую johnvansickle ffmpeg static build (для VPS без compilation). В нём **нет фильтра drawtext** — text не отрендерить нативно.

Решение: **Pillow рендерит PNG с текстом**, ffmpeg делает overlay:

```python
# Pillow: создаём прозрачный PNG с текстом
img = Image.new("RGBA", (box_w, box_h), bg_color)
draw = ImageDraw.Draw(img)
draw.text(..., font=ImageFont.truetype(FONT_BOLD, 64))
img.save(out_path)

# ffmpeg: overlay PNG на видео
subprocess.run([
    "ffmpeg", "-i", source, "-i", text_png,
    "-filter_complex", "[0][1]overlay=(W-w)/2:H-h-280",
    output
])
```

Это даёт **полный контроль над типографикой** (шрифт, цвет, скруглённые углы плашки) без переборок ffmpeg.

### 5. Layout safe-zones для video Shorts

После 4 первых публикаций в YouTube Shorts я понял что текст-плашки нельзя кидать абы куда — они **наложатся на лицо** и снизят engagement (мне пришлось перезаливать 4 видео).

Правила, к которым пришёл:
- Заголовок-хук — **сверху по центру**, отступ 8-10% от верхнего края (y=160 при 1920px высоте)
- Watermark — **правый нижний угол** с отступом 30px от обоих краёв
- Заголовок поста — **внизу**, поднят на 280px от низа чтобы не пересекался с watermark
- Все плашки с непрозрачным фоном (alpha=200/255)

Это значительно лучше для удержания внимания.

## Юнит-экономика

- 1 видео через Kling = $0.10 (~10₽)
- Storage + CDN = ~0.5₽ на видео (Timeweb VPS 1Тб трафика бесплатно)
- Тариф «Микро» = 99₽ за 1 видео → **маржа 88₽ (89%)**

При среднем чеке 290₽ за пакет из 10 видео и реальном использовании ~3-5 видео из пакета — фактическая маржа выходит ~250₽ с подписки.

## Что бы я сделал иначе на старте

1. **Не делать собственный лендинг** — Telegram-бот конвертирует в 5-10 раз лучше. На лендинге 0 платежей с улицы за 3 дня, в боте — 200+ за месяц.
2. **Сразу делать sticky-кнопку «Купить» на мобильном** — без неё на 12000px высоте лендинга юзер просто не доходит до тарифов.
3. **Прогревать пользователя email/TG-cascade-ом** до показа тарифов. Сначала эмоции, потом продажа.

## Заключение

Kling 2.5 + PiAPI — **продакшн-готовое сочетание** для русского B2C на оживлении фото. Sora и Runway проигрывают по цене и доступу, но выигрывают по реалистичности на сложных сценариях (танец, бег, сложные позы).

Для эмоциональной ниши «оживить фото бабушки» Kling 2.5 — **достаточен**. Главная проблема не в технологии, а в **конверсии лендинга**, что лежит уже в UX/маркетинге, а не в нейросетях.
